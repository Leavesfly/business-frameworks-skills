## 三、如何触发与使用

### 3.1 直接提问（推荐）
任何自然语言提问，Claude 会自动激活 `business-frameworks-router` 主 skill 进行分诊：

- "帮我分析一下 XX 市场"
- "这个产品该怎么定位？"
- "团队执行力差怎么办？"
- "给老板汇报这个项目"

### 3.2 显式调用子 skill
如果已明确知道要用哪一章，可直接点名：

- "用 ch4 的商业模式图帮我梳理一下这个生意"
- "用 ch1 的逻辑树状图拆一下这个问题"

### 3.3 显式调用具体框架
也可以直接点名某个框架（70 个均可）：

- "帮我做一版 SWOT"、"做个 RACI 矩阵"、"用 GROW 模型辅导我一次"

## 六、每个 Skill 的统一结构

每个子 skill 的 `SKILL.md` 都遵循：

1. **YAML frontmatter**：`name` + `description`（含中英触发关键词）
2. **定位**：本章在 MAP 循环中的位置
3. **与其他章的关系**：上下游
4. **框架清单**：每个框架含
   - 用途（何时用）
   - 步骤 / 提问清单
   - 可直接填写的模板
   - 英文关键词（便于 Claude 匹配英文提问）
5. **常用组合**：章内框架的典型串联
6. **输出契约**：本章必须产出的交付物

## 七、维护与扩展

- **新增框架**：在对应 `chX-*/SKILL.md` 的"框架清单"里追加条目，保持模板结构一致
- **新增章节**：新建 `chN-*/SKILL.md` 目录，并更新 `business-frameworks-router/SKILL.md` 中的子 skill 目录表
- **跨章组合**：在对应子 skill 末尾的"常用组合"或主 skill 的"Recipes"中登记

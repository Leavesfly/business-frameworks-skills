# 第 1 章 · 发现问题、课题（Problem Discovery）

**定位**：小循环① 的起点。当你不清楚"真正要解决的问题是什么"时，用本章框架把模糊的抱怨/现象 → 精炼成一句可执行的课题陈述。

**与其他章的关系**：
- ⇄ `ch2-market-analysis`（小循环①：边发现边分析）
- → `ch3-solution-ideation`（问题定义清楚后进入方案构思）

---

## 框架清单（01–08）

### ① As is / To be（现状–理想差距法）
- **用途**：用"现状 vs 理想"的落差暴露问题。
- **适用场景**：新人接手、战略起点、问题模糊。
- **步骤**：
  1. 描述 **As is**（当前事实，不含评价）
  2. 描述 **To be**（理想状态，可量化）
  3. 标出 **Gap**（差距的 3 个关键维度）
- **模板**：
  ```
  | 维度 | As is 现状 | To be 理想 | Gap 差距 |
  |------|-----------|-----------|---------|
  |      |           |           |         |
  ```
- **英文关键词**：as-is to-be, current vs target, gap analysis

### ② 6W2H
- **用途**：用 8 个疑问词拆解一件事，避免遗漏。
- **适用场景**：接到模糊需求、写立项文档。
- **步骤**：逐个回答 What / Why / Who / Whom / When / Where / How / How much。
- **模板**：
  ```
  - What  做什么：
  - Why   为什么做：
  - Who   谁来做：
  - Whom  做给谁：
  - When  何时做：
  - Where 在哪做：
  - How   怎么做：
  - How much 预算/成本：
  ```
- **英文关键词**：6W2H, 5W1H, scoping questions

### ③ 原因分析（Root Cause Analysis / 5 Whys）
- **用途**：连问 5 次"为什么"直达根因。
- **步骤**：现象 → Why1 → Why2 → … → Why5 → 根因
- **模板**：
  ```
  现象：___
  Why 1：因为 ___
  Why 2：因为 ___
  Why 3：因为 ___
  Why 4：因为 ___
  Why 5：因为 ___（根因）
  ```
- **英文关键词**：5 whys, root cause, fishbone

### ④ 可控制 / 不可控制（Control / Uncontrol 矩阵）
- **用途**：把精力聚焦在自己能影响的事情上。
- **步骤**：列出所有相关因素 → 分类到"可控 / 不可控" → 只对可控因素制定行动；不可控因素只做"监测 + 预案"。
- **模板**：
  ```
  ┌─────────────┬─────────────┐
  │ 可控制       │ 不可控制     │
  │ (行动清单)  │ (监测/预案)│
  │ - ...       │ - ...       │
  │ - ...       │ - ...       │
  └─────────────┴─────────────┘
  ```
- **英文关键词**：circle of control, controllable factors, Stephen Covey

### ⑤ 逻辑树状图（Logic Tree / Issue Tree / MECE）
- **用途**：把一个大问题拆成 MECE（不重不漏）的子问题。
- **三种树**：What 树（拆构成） / Why 树（究原因） / How 树（批解行动）
- **步骤**：
  1. 顶层：核心问题一句话
  2. 第二层：3–5 个主分支（遵循 MECE）
  3. 第三层：继续展开至可行动的叶子节点
  4. MECE 自检：同一层的分支能否相加还原为顶层？是否有交叉？
- **模板**：
  ```
  核心问题
  ├─ 分支 A
  │  ├─ A1
  │  └─ A2
  ├─ 分支 B
  │  ├─ B1
  │  └─ B2
  └─ 分支 C
  ```
- **英文关键词**：logic tree, issue tree, MECE, hypothesis tree, pyramid principle

### ⑥ 课题设定表单（Issue Setting Sheet）
- **用途**：把混乱思考**一页纸**定格为可执行的课题。
- **模板**：
  ```
  【背景】：
  【现状】：
  【理想】：
  【差距】：
  【课题陈述（一句话）】：
  【成功标准】：
  【截止时间】：
  【责任人】：
  ```
- **英文关键词**：issue brief, problem statement

### ⑦ 急迫性 / 重要性矩阵（Urgent × Important，艾森豪威尔矩阵）
- **用途**：对一堆任务/问题排优先级。
- **模板**：
  ```
                急迫
               ↑
    重要·急迫  │ 重要·不急迫
    （立刻做） │ （计划做）
  ─────────────┼─────────────→ 重要
    不重要·急迫│ 不重要·不急迫
    （授权）   │ （删除）
               │
  ```
- **英文关键词**：Eisenhower matrix, urgency importance

### ⑧ 决策矩阵（Decision Matrix, Weighted Scoring）
- **用途**：多个候选方案之间的定量比较。
- **步骤**：
  1. 列出候选方案（行）
  2. 列出评估维度（列），给每个维度权重（合计 100%）
  3. 每个方案在每个维度打分（1–5）
  4. 加权求和，得分最高者胜出
- **模板**：
  ```
  | 方案 \ 维度 | 成本(30%) | 收益(40%) | 风险(30%) | 加权得分 |
  |-------------|-----------|-----------|-----------|---------|
  | 方案 A      |    4      |    3      |    5      |  3.9    |
  | 方案 B      |    3      |    5      |    3      |  3.8    |
  ```
- **英文关键词**：decision matrix, weighted scoring, pugh matrix

## 常用组合

- **问题完全模糊** → `As is/To be`① + `6W2H`② + `课题设定表单`⑥
- **问题大而复杂** → `逻辑树状图`⑤ + `原因分析`③
- **问题一堆分不清主次** → `急迫性/重要性矩阵`⑦ + `决策矩阵`⑧
- **资源有限焦虑感强** → `可控制/不可控制`④

## 输出契约

完成本章后必须产出：
1. **一句话的课题陈述**
2. **根因或主矛盾**
3. **建议下一步**：进入 `ch2-market-analysis`（补充事实）或 `ch3-solution-ideation`（已清晰，开始想方案）

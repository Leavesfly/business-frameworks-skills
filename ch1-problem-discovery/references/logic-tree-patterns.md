## MECE 自检清单

1. **相加还原**：同层分支相加能否完全覆盖上层节点？
2. **无交叉重叠**：同层分支之间是否有内容重复或边界模糊？
3. **层级一致**：同层分支是否处于相同抽象粒度？
4. **穷尽可能**：是否遗漏重要分支维度？（可用"其他"兜底）
5. **独立完整**：每个分支能否独立表达一个子问题？

## 常见反模式

### ❌ 错误 1：分类标准混乱
```
用户群体
├─ 年轻用户        ← 按年龄
├─ 高消费用户      ← 按消费
├─ 北京用户        ← 按地域
```
**问题**：三个维度混用，严重重叠。应统一按单一维度拆分。

### ❌ 错误 2：层级跳跃
```
销售额下降
├─ 市场因素
│  └─ 竞争对手降价 5% 导致我们丢失 3 个大客户
```
**问题**：第二层直接跳到具体事件，缺少中间层级。应先拆"竞争影响→客户流失→大客户丢失"。

### ❌ 错误 3：遗漏重要分支
```
APP 日活下降
├─ 新用户减少
└─ 老用户流失
```
**问题**：忽略了"用户使用频次降低但仍在活跃"的情况。应补充"活跃用户 engagement 下降"。

### ❌ 错误 4：分支过多过细
```
成本结构
├─ 办公用品费
│  ├─ 笔
│  ├─ 纸
│  └─ ...（列出 50+ 项）
```
**问题**：叶子节点过于琐碎。应聚合为"办公耗材"并在第三层展开。

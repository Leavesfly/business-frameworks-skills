# 填好样例 · Problem Discovery Worked Examples

> `ch1-problem-discovery` 的扩展参考。用已填好的真实案例演示每个框架的输出形态。
> Agent 在引导用户填写模板时，可对照此处的"合格示范"把控深度。

---

## 示例场景：某 SaaS 产品续约率下降

### 原始诉求
"我们的 SaaS 产品最近 3 个月续约率从 85% 跌到 68%，老板很着急，让我们尽快解决，可是我也不知道从哪入手。"

---

## ① As is / To be 样例

| 维度 | As is 现状 | To be 理想 | Gap 差距 |
|---|---|---|---|
| 年续约率 | 68%（近 3 月） | 85%（回到基线） | -17pp |
| NPS | 32 | ≥ 45 | -13 |
| 客户成功团队人数 | 3 人 / 120 客户 | 1:30 配比 = 4 人 | 缺 1 人 |
| 产品健康度打分机制 | 无 | 每周自动跑分 + 高危预警 | 从 0 到 1 |

**一句话差距**：续约率从 85% 跌到 68%，主要来自"高危客户没有被及时识别和干预"。

---

## ② 6W2H 样例

- **What**：把续约率从 68% 拉回 85%
- **Why**：续约是 SaaS LTV 的核心，每 1pp 价值约 120 万 ARR
- **Who**：CS 团队主导，产品 & 销售配合
- **Whom**：续约窗口前 90 天的 A/B 类客户（共 46 家）
- **When**：3 个月内见效，6 个月回到基线
- **Where**：核心干预阵地 = 客户线上使用数据 + 季度 QBR 会议
- **How**：建立健康度打分 → 分层运营 → 周级复盘
- **How much**：预算 30 万（人力 + 系统开发），预期回收 3600 万 ARR

---

## ③ 原因分析（5 Whys）样例

- **现象**：续约率 3 个月跌 17pp
- **Why 1**：因为有 15 家 A/B 类客户主动流失
- **Why 2**：因为他们在续约前 60 天就停止了核心功能使用
- **Why 3**：因为没人主动联系他们了解问题
- **Why 4**：因为 CS 只盯"即将到期"的客户，没有预警机制
- **Why 5（根因）**：因为产品和 CS 之间缺少"客户健康度"的共享数据口径

→ **结论**：根因是"健康度信号系统缺失"，而不是"客服不够努力"。

## ④ 可控 / 不可控 样例

```
┌──────────────────────────┬──────────────────────────┐
│ 可控制 (行动清单)         │ 不可控制 (监测/预案)       │
├──────────────────────────┼──────────────────────────┤
│ - 健康度打分模型          │ - 宏观经济紧缩             │
│ - CS 人力配置             │ - 竞对降价策略             │
│ - 产品使用数据看板         │ - 客户自身业务萎缩          │
│ - QBR 会议流程            │ - 行业预算收紧             │
└──────────────────────────┴──────────────────────────┘
```

**聚焦**：只对左侧 4 项制定 SMART 行动；右侧写一份《高风险预警预案》即可。

## ⑤ 逻辑树状图（How 树）样例

```
如何把续约率拉回 85%
├─ 预警识别（早发现）
│  ├─ 健康度打分模型（产品 + CS 联合设计）
│  └─ 高危客户周报自动推送
├─ 主动干预（快响应）
│  ├─ CS 主动回访 SOP（高危 7 天内）
│  └─ 产品专家介入（技术问题专项）
└─ 关系深化（防复发）
   ├─ 季度 QBR 会议标准化
   └─ 高管 Sponsor 对接 Top 20 客户
```

**MECE 自检**：三大分支 "预警→干预→深化" 对应时间轴，相加即完整生命周期 ✓

## ⑥ 课题设定表单样例

```
【背景】：SaaS 产品 2025 Q4 续约率从 85% 跌至 68%
【现状】：15 家 A/B 类客户主动流失；CS 团队被动响应；无健康度数据
【理想】：续约率回到 85%+；建立主动预警体系；CS 服务比 1:30
【差距】：-17pp 续约率；缺少健康度系统；CS 缺 1 人
【课题陈述（一句话）】：
  在 6 个月内，通过建立"健康度预警 + 主动干预"体系，
  把 SaaS 产品续约率从 68% 恢复到 85%。
【成功标准】：
  1. 续约率 ≥ 85%（月度滚动）
  2. 高危客户预警提前 ≥ 45 天
  3. CS 主动触达率 ≥ 90%
【截止时间】：2026-06-30
【责任人】：CS 总监（主） / 产品经理（协同）
```

## ⑧ 决策矩阵样例

**候选方案**：如何优先投入续约提升？

| 方案 \ 维度 | 见效速度(30%) | ARR 影响(40%) | 实施难度(30%) | 加权得分 |
|---|---|---|---|---|
| A. 建健康度系统 | 3 | 5 | 2 | **3.5** |
| B. 扩招 CS | 4 | 3 | 4 | **3.6** |
| C. Top 客户高管对接 | 5 | 4 | 5 | **4.6** ⭐ |
| D. 产品专家驻场 | 2 | 3 | 3 | **2.7** |

→ **先启动 C**（Top 客户高管对接，本季度见效），**并行启动 A**（长期机制）。

## 常见失败模式

| 反模式 | 正确做法 |
|---|---|
| 只有"现状"没有"理想" | 必须量化 To be 才能算 Gap |
| 5 Whys 问成"都是外部原因" | 每个 Why 都必须有内部可控项 |
| 逻辑树只有两层 | 至少拆到团队当周可开工的叶子 |
| 课题陈述超过 3 行 | 必须一句话，否则说明还没想清楚 |
| 决策矩阵维度只有 2 个 | 至少 3 个维度才能避免单维偏误 |

# 第 6 章 · 管理组织（Organization Management）

**定位**：支援性活动。处理"人"的问题 —— 方向对齐、动力激发、能力培养、关系协调。

## 框架清单（56–66）

### 56 任务 / 愿景 / 价值（Mission / Vision / Values, MVV）
- **用途**：定义组织"为何存在、要去哪里、靠什么"。
- **三要素**：
  - **Mission 任务/使命**：我们存在的理由（为谁创造什么价值）
  - **Vision 愿景**：未来 3–10 年想成为的样子
  - **Values 价值观**：做决策和行为的准则
- **模板**：
  ```
  Mission（使命）：为【X】提供【Y】，让他们【Z】。
  Vision（愿景）：到【年份】，我们将成为【描述】。
  Values（价值观）：
    1. ___（行为准则）
    2. ___
    3. ___
  ```
- **英文关键词**：mission vision values, MVV, purpose statement

### 57 Will / Can / Must（意愿 / 能力 / 必须）
- **用途**：找到工作的"甜蜜点"，也可用于个人发展与岗位匹配。
- **三圈**：
  - **Will** 想做的（热情所在）
  - **Can** 能做的（能力所及）
  - **Must** 必须做的（组织/市场要求）
- **三圈交集 = 最佳发力点**
- **模板**：
  ```
      Will（想做）
        ╱─╲
       ╱   ╲
      ╱ ★  ╲
  Can────────Must
  （能做）   （必须）
  ```
  每圈列 3–5 项，标出交集部分。
- **英文关键词**：will can must, sweet spot, Ikigai 相近

### 58 Need / Want 矩阵
- **用途**：区分客户/员工/相关方"真正需要的"vs"表面想要的"，避免做错需求。
- **两维**：Need（实际需要，高/低）× Want（表面想要，高/低）
- **模板**：
  ```
            需要 Need 高
               │
    隐性需求   │   核心需求
    （挖掘）   │   （必做）
  ─────────────┼───────────── 想要 Want 高
    伪需求     │   自我陶醉需求
    （删掉）   │   （警惕）
               │
            需要 Need 低
  ```
- **英文关键词**：need want matrix, jobs to be done

### 59 周哈里窗（Johari Window）
- **用途**：理解"自己 vs 他人"的认知差异，扩大公开区、缩小盲区。
- **四区**：
  ```
               | 自己知道    | 自己不知道   |
  -------------+-------------+--------------|
  他人知道     | 公开区 Open | 盲区 Blind   |
  他人不知道   | 隐藏区 Hidden | 未知区 Unknown |
  ```
- **使用**：通过**自我揭露**缩小隐藏区，通过**反馈**缩小盲区。
- **英文关键词**：Johari window, self awareness, feedback

### 60 认知 / 行动循环（OODA Loop）
- **用途**：在快速变化的环境中做决策和反应。
- **四步循环**：
  - **O**bserve 观察
  - **O**rient 定位（分析、解读）
  - **D**ecide 决策
  - **A**ct 行动 → 回到 Observe
- 谁的 OODA 环跑得更快，谁就掌握主动权。
- **英文关键词**：OODA loop, Boyd cycle, decision cycle

### 61 Want / Commitment（想要 × 承诺 矩阵）
- **用途**：团队目标管理时区分"嘴上说想要"与"愿意为之付出"。
- **两维**：Want（想要程度 高/低）× Commitment（承诺行动 高/低）
- **四象限**：
  ```
              承诺 Commitment 高
                │
    挂嘴边但不做 │  真行动者
    （引导）    │  （授权）
  ──────────────┼──────────── 想要 Want 高
    随波逐流    │  抱怨者
    （激发）    │  （促兑现）
                │
              承诺 Commitment 低
  ```
- **英文关键词**：want commitment matrix, accountability

### 62 PM 理论（三隅二不二的 P-M 领导理论）
- **用途**：领导行为的二维评估。
- **两维**：
  - **P**erformance 目标达成功能（Performance function）：推动业绩、布置任务、检查进度
  - **M**aintenance 团队维持功能（Maintenance function）：关心成员、调和关系、营造氛围
- **四类领导**：
  ```
              M 团队维持 高
                │
       pM       │      PM
     （人缘型） │    （理想型）
  ──────────────┼──────────── P 目标达成 高
       pm       │      Pm
     （放任型） │    （业绩型）
                │
              M 团队维持 低
  ```
- **英文关键词**：PM theory, Misumi leadership, performance maintenance

### 63 利害关系人分析（Stakeholder Analysis）
- **用途**：识别项目相关方，按"权力 × 关注度"制定沟通策略。
- **步骤**：
  1. 列出所有相关方（内外部）
  2. 评估每方的 **Power 权力** 和 **Interest 关注度**
  3. 定位到四象限，制定策略
- **模板**：
  ```
            权力 Power 高
                │
    让其满意    │   紧密管理
    （关键人）  │   （核心干系人）
  ──────────────┼──────────── 关注 Interest 高
    最小投入    │   保持告知
                │
            权力 Power 低
  ```
  附清单表：
  ```
  | 干系人 | 权力 | 关注 | 立场(支持/中立/反对) | 沟通策略 |
  |--------|------|------|---------------------|---------|
  ```
- **英文关键词**：stakeholder analysis, power interest grid, RACI 互补

### 64 双因素理论（Herzberg 保健 × 激励）
- **用途**：区分"给了不爽、不给不爽、给了就爽"的因素，正确设计激励。
- **两类因素**：
  - **保健因素 Hygiene**：工资、工作条件、制度、人际、上司 — 不足会不满，足够≠满意
  - **激励因素 Motivator**：成就、认可、责任、成长、工作本身 — 有则满意，无则平淡
- **模板**：
  ```
  | 类别 | 因素 | 当前水平 | 改进方向 |
  |------|------|---------|---------|
  | 保健 | 工资 |         |         |
  | 保健 | 制度 |         |         |
  | 激励 | 认可 |         |         |
  | 激励 | 成长 |         |         |
  ```
- **英文关键词**：Herzberg two factor, hygiene motivator

### 65 Will / Skill 矩阵（意愿 × 能力，情境领导力矩阵）
- **用途**：对下属按"意愿 × 能力"分类，采取不同的领导方式（Landsberg 经典模型）。
- **四象限 & 领导策略**：
  ```
              意愿 Will 高
                │
    Guide 指导 │   Delegate 授权
    （高意愿     │   （明星员工）
     低能力）   │
  ─────────────┼──────────── 能力 Skill 高
    Direct 指派  │   Excite 激励
    （低意愿     │   （老兵丧失
     低能力）   │    斗志：高能
                │    力低意愿）
              意愿 Will 低
  ```
  - **高 Will + 高 Skill**：**Delegate 授权**，放手交给他
  - **高 Will + 低 Skill**：**Guide 指导**，辅导能力成长
  - **低 Will + 高 Skill**：**Excite 激励**，重新点燃动力
  - **低 Will + 低 Skill**：**Direct 指派**，明确步骤严监控，或重新评估岗位匹配
- **英文关键词**：skill will matrix, situational leadership, Landsberg, Hersey Blanchard

### 66 GROW 模型（辅导对话四步法）
- **用途**：一对一辅导、Coaching 对话的结构。
- **四步**：
  - **G**oal 目标：你想达成什么？什么样算成功？
  - **R**eality 现状：现在情况怎样？有哪些事实？
  - **O**ptions 选项：你有哪些可能的做法？各自的优劣？
  - **W**ill 行动意愿：你决定做什么？第一步是？何时做？
- **模板（提问清单）**：
  ```
  G: 这次谈完希望达成什么？三个月后的目标？
  R: 现状是什么？你做过什么尝试？还差什么？
  O: 有哪些选项？每个的好处 / 代价？还有别的可能吗？
  W: 你决定先做哪一步？什么时候做？需要什么支持？
  ```
- **英文关键词**：GROW model, coaching conversation, Whitmore

## 常用组合

- **团队方向不明** → `任务愿景价值`56 → `GROW`66（为团队定目标）
- **招聘 / 岗位匹配** → `Will/Can/Must`57 + `Will/Skill 矩阵`65
- **真伪需求** → `Need/Want 矩阵`58
- **沟通破冰** → `周哈里窗`59
- **目标执行不力** → `Want/Commitment`61 + `Will/Skill`65 + `GROW`66
- **领导风格调整** → `PM 理论`62 + `Will/Skill`65
- **项目/变革管理** → `利害关系人分析`63 + `OODA`60
- **激励机制设计** → `双因素理论`64 + `Want/Commitment`61

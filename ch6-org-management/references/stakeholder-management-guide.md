## 1. 利害关系人分析案例

### 案例 1：内部系统改造项目

#### 干系人识别清单

| 干系人 | 角色 | 权力 | 关注 | 立场 | 沟通策略 |
|--------|------|------|------|------|---------|
| CTO | 决策者 | 高 | 高 | 支持 | 紧密管理，每周汇报进展 |
| 业务部门负责人 | 使用者 | 中 | 高 | 中立 | 让其满意，定期演示功能 |
| IT 运维团队 | 配合方 | 中 | 中 | 中立 | 保持告知，邀请参与技术方案评审 |
| 一线员工 | 最终用户 | 低 | 高 | 担忧 | 保持告知，收集反馈优化体验 |
| 外部供应商 | 实施方 | 低 | 中 | 支持 | 最小投入，按合同管理 |

#### Power/Interest 定位图
```
            权力 Power 高
                │
    CTO         │   CTO(紧密管理)
   (让其满意)   │
  ──────────────┼──────────── 关注 Interest 高
   运维团队     │   业务负责人
   (保持告知)   │   (让其满意)
                │
            权力 Power 低
```

## 2. Want/Commitment 矩阵案例：跨部门项目诊断

### 场景：公司数字化转型项目，涉及产品、技术、运营、市场四部门

| 部门 | Want（想要程度） | Commitment（承诺行动） | 定位 | 应对策略 |
|------|-----------------|---------------------|------|---------|
| 产品部 | 高 | 高 | 真行动者 | 授权主导，给予资源 |
| 技术部 | 高 | 低 | 抱怨者 | 明确排期约束，促兑现承诺 |
| 运营部 | 低 | 高 | 随波逐流 | 激发兴趣，展示价值 |
| 市场部 | 低 | 低 | 旁观者 | 暂不强求，保持信息同步 |

**结论**：重点解决技术部的资源冲突问题，同时向运营部展示转型对其业务的实际帮助。

## 3. OODA 循环案例：竞对突然降价时的快速决策

### 场景：竞争对手周一早上宣布核心产品降价 30%

| 阶段 | 行动 | 耗时 |
|------|------|------|
| **Observe** 观察 | 监控到竞品公告、收集媒体报道、监测用户咨询量激增 | 2 小时 |
| **Orient** 定位 | 分析：对方清库存？试探市场？评估对我方影响：预计流失 15% 价格敏感客户 | 3 小时 |
| **Decide** 决策 | 方案 A：跟随降价；方案 B：强调差异化价值；方案 C：推出限时优惠包 → 选择 B+C 组合 | 2 小时 |
| **Act** 行动 | 发布官方回应强调服务优势、上线老客户专属优惠、销售团队统一话术培训 | 当天完成 |

**结果**：次日用户咨询恢复正常，一周后数据显示仅流失 8% 客户，优于预期。

**关键**：整个 OODA 循环在 1 个工作日内完成，快于竞品预期的市场反应时间。

## 4. Need/Want 矩阵案例：内部需求管理

### 场景：收到各部门提出的 20+ 系统改进需求

| 需求 | Need（实际需要） | Want（表面想要） | 象限 | 处理 |
|------|-----------------|-----------------|------|------|
| 报表导出 Excel | 高 | 高 | 核心需求 | 必做，优先排期 |
| 界面改成深色模式 | 低 | 高 | 自我陶醉 | 警惕，放入 backlog |
| 增加数据权限控制 | 高 | 低 | 隐性需求 | 挖掘，主动提出方案 |
| 每个页面加动画 | 低 | 低 | 伪需求 | 删掉，礼貌拒绝 |

**四象限定位图**：
```
            Need 高
              │
  隐性需求    │   核心需求
  (主动挖掘)  │   (立即执行)
  ────────────┼──────────── Want 高
  伪需求      │   自我陶醉
  (坚决拒绝)  │   (延后考虑)
              │
            Need 低
```

**方法论**：通过访谈挖掘"隐性需求"背后的真实痛点，对"自我陶醉需求"要求提出者证明业务价值。

## 5. 「难搞干系人」应对策略清单

| 类型 | 特征 | 应对方法 |
|------|------|---------|
| **掌控型** | 强势、喜欢指挥、不接受异议 | 给足面子，私下沟通；用数据说话；让其感觉是"他的主意" |
| **拖延型** | 永远说"再看看"、不表态 | 设定明确 deadline；升级求助其上级；拆解小步骤降低决策压力 |
| **挑剔型** | 永远不满意、找茬 | 提前确认验收标准；书面记录每次反馈；邀请其参与过程而非只看结果 |
| **回避型** | 不回消息、不参加会议 | 换沟通渠道（电话/当面）；找到其信任的中间人；明确不参与的后果 |
| **情绪型** | 易怒、抱怨多、负能量 | 先倾听共情，再谈事；避免正面冲突；必要时暂停对话，冷静后再谈 |

**通用原则**：
1. 了解对方的 KPI 和压力来源
2. 找到共同利益点
3. 保持专业，不被情绪带偏
4. 必要时向上升级，寻求支持

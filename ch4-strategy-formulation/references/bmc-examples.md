# BMC 行业样例 · Business Model Canvas Examples

> `ch4-strategy-formulation` 的扩展参考。提供 3 个不同行业的 BMC 填写范例，展示九格如何在真实业务中相互呼应。

## 范例一：SaaS（HR 员工体验平台）

```
┌──────────┬──────────┬──────────────────┬──────────┬──────────┐
│ KP 关键合作 │ KA 关键业务 │ VP 价值主张       │ CR 客户关系 │ CS 客户细分 │
│            │            │                 │            │            │
│ - 钉钉/飞书 │ - 产品研发  │ 让 1000 人以下   │ - KA 成功   │ - 200-1000 │
│   集成商    │ - 客户成功  │ 企业 10 分钟     │ - 社区运营  │   人成长期 │
│ - HR SaaS   │ - 标杆客户  │ 搭出完整员工     │ - 年度 QBR  │   科技公司 │
│   联盟      │   打磨     │ 体验体系         │            │ - HRBP/CHO│
│ - 云厂商    ├──────────┤ （入职/假期/     ├──────────┤ - 行业：  │
│             │ KR 关键资源 │  调薪/离职）     │ CH 渠道    │   互联网/ │
│             │ - 工程团队  │                 │ - SaaS 电销│   消费/  │
│             │ - 行业数据  │ 对比 Workday：   │ - 伙伴转介 │   生活服务│
│             │ - 客户案例  │ 轻、便宜 1/10    │ - 社区内容 │            │
├──────────┴──────────┴──────────────────┴──────────┴──────────┤
│ C$ 成本结构                          │ R$ 收入来源                    │
│ - 研发工资（45%）                    │ - 订阅费：500/人/年             │
│ - 云服务（15%）                      │ - 实施费：按模块 5-30 万一次性  │
│ - 市场 + 销售（25%）                  │ - 增值模块：绩效/培训/OKR       │
│ - 客户成功（15%）                     │ - 合作伙伴分成（5% 渠道佣金）   │
└──────────────────────────────────────────────────────────────┘
```

**验证问题**：
- VP 是否够痛？→ 测试 HRBP 的续费意愿（>90% 算过关）
- CS 是否够集中？→ 200-1000 人科技公司市场池约 3 万家，足够支撑 1000 家付费
- R$ 是否够深？→ ARPU 20 万 + 续约 90% = LTV 200 万（对比 CAC 10 万，健康）

## 范例二：餐饮连锁（国产精品咖啡下沉县城）

```
┌──────────┬──────────┬──────────────────┬──────────┬──────────┐
│ KP 关键合作 │ KA 关键业务 │ VP 价值主张       │ CR 客户关系 │ CS 客户细分 │
│            │            │                 │            │            │
│ - 云南豆源 │ - 门店运营  │ 县城里的那家     │ - 会员小程序│ - 县城     │
│   合作社    │ - 加盟督导  │ 「我的咖啡馆」   │ - 社群运营  │   24-35 岁 │
│ - 本地物流 │ - 烘焙中心  │                 │ - 储值卡   │   小镇青年 │
│ - 加盟商    │ - 选址     │ · 15 元精品豆   ├──────────┤ - 县中学    │
│ - 小红书    ├──────────┤ · 县城首店    │ CH 渠道    │   老师/    │
│   KOC      │ KR 关键资源 │ · 30 座小而美   │ - 门店     │   医院护士 │
│            │ - 云南产区  │ · 拍照级颜值     │ - 小红书   │ - 县城    │
│            │ - 加盟网络  │                 │ - 美团外卖 │   白领    │
│            │ - 品牌 IP   │                 │            │            │
├──────────┴──────────┴──────────────────┴──────────┴──────────┤
│ C$ 成本结构                          │ R$ 收入来源                    │
│ - 门店租金 + 人工（35%）              │ - 堂食 55%                    │
│ - 原材料（30%）                       │ - 外卖 30%                    │
│ - 加盟督导 + 总部（15%）               │ - 加盟初装费（一次性 8 万）    │
│ - 营销（10%，重压小红书）              │ - 原料供应链（对加盟商 +15%）  │
│ - 其他（10%）                         │ - 会员储值（现金流）           │
└──────────────────────────────────────────────────────────────┘
```

**关键洞察**：R$ 不止"卖咖啡"，真正盈利来自加盟费 + 原料供应 + 会员沉淀金。VP 必须呼应 CS 的情感需求（"我的咖啡馆"，而非便宜咖啡）。

## 范例三：双边平台（自由职业对接平台）

```
┌──────────┬──────────┬──────────────────┬──────────┬──────────┐
│ KP 关键合作 │ KA 关键业务 │ VP 价值主张       │ CR 客户关系 │ CS 客户细分 │
│            │            │                 │            │            │
│ - 认证机构 │ - 匹配算法  │ 企业侧：         │ - 自助平台  │ 需求侧：   │
│ - 支付通道 │ - 人才库运营 │ 3 天内对接      │ - 客户成功  │ 中小企业   │
│ - 律师 / 税 │ - 客户成功  │ 可验证的自由    │   (KA)     │ 项目 PM    │
│   代记账   ├──────────┤ 职业者（设计/   ├──────────┤            │
│ - 社群大 V │ KR 关键资源 │ 开发/内容）     │ CH 渠道    │ 供给侧：   │
│            │ - 双边流量 │                 │ - SEO/SEM  │ 成熟自由   │
│            │ - 信用体系 │ 人才侧：         │ - 社群     │ 职业者     │
│            │ - 争议仲裁 │ 稳定接单 + 自动 │ - 转介绍    │ （3 年+）  │
│            │            │ 开票 + 争议保障 │            │            │
├──────────┴──────────┴──────────────────┴──────────┴──────────┤
│ C$ 成本结构                          │ R$ 收入来源                    │
│ - 算法/技术团队                       │ - 交易佣金（10-15%）           │
│ - 运营 + 仲裁                         │ - 认证费（人才侧）             │
│ - 市场拉新（双边）                    │ - 会员订阅（企业侧，找人无限 ）│
│ - 风控 + 法务                         │ - 增值：代记账 / 合同模板     │
└──────────────────────────────────────────────────────────────┘
```

**双边平台要点**：CS / VP / CH 都要"两套"，并且要判断先做哪一边（Chicken-Egg 问题）。本例：先用高额补贴吸引人才侧，再用"3 天找到人"吸引企业侧。

## 九格填写顺序（推荐）

```
1 CS (Customer Segments)       — 先想清楚服务谁
2 VP (Value Proposition)       — 对他们有什么不可替代的价值
3 CH (Channels)                — 怎么触达
4 CR (Customer Relationships)  — 维持什么样的关系
5 R$ (Revenue Streams)         — 如何收钱
6 KR (Key Resources)           — 需要什么资源才能交付
7 KA (Key Activities)          — 要做什么核心动作
8 KP (Key Partners)            — 谁帮你做
9 C$ (Cost Structure)          — 最后看成本
```

## BMC 自检清单

- [ ] CS 是否具体到"可描述的人"而非"模糊的人群标签"？
- [ ] VP 是否一句话能讲清，且不可替代？
- [ ] VP 与 CS 的痛点直接对应？
- [ ] R$ 是否多元（至少 2 个收入来源）？
- [ ] C$ 的最大头 ≥ 50% 时是否风险可控？
- [ ] KR 中是否有"难以被竞对复制"的资源？
- [ ] 九格是否"闭环"：右侧 4 格（CS/CR/CH/VP）收钱，左侧 4 格（KP/KA/KR/C$）花钱，中间 VP 连通？

## 常见反模式

| 反模式 | 症状 | 正确做法 |
|---|---|---|
| VP 写成功能列表 | "功能 A、功能 B、功能 C" | 一句话讲清"为谁解决了什么"|
| CS 过宽 | "所有中小企业" | 必须圈到可数的几千到几万家 |
| R$ 只有一列 | 只卖硬件不卖服务 | 找出潜在的 2-3 条收入线 |
| 九格之间互不呼应 | VP 说精品，C$ 在压成本 | 每次画完做一次"因果链"验证 |
| 画完就存档 | 从不迭代 | BMC 是活文档，每季度评审一次 |

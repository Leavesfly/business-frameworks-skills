## 一、STP完整案例

### 案例1：新茶饮品牌定位

**背景**：某新茶饮品牌在二线城市开设首店，需在竞争激烈的市场中找到差异化定位。

**S - Segmentation（市场细分）**：
- **人口统计维度**：18-25岁学生、26-35岁白领、35岁以上家庭客群
- **消费行为维度**：价格敏感型（15元以下）、品质追求型（15-25元）、体验导向型（25元以上+社交空间）
- **心理维度**：健康养生派（低糖/鲜果）、潮流打卡派（高颜值/联名）、功能需求派（提神/解腻）

**T - Targeting（目标选择）**：
选择**26-35岁白领中的品质追求型+健康养生派**作为核心目标客群，理由：
1. 该群体消费频次高（每周2-3次），客单价稳定在20元左右
2. 对"低糖""鲜果""真奶"等健康标签敏感，愿意为品质溢价
3. 社交媒体活跃度高，易形成口碑传播
4. 避开学生群体的价格战和高端客群的 niche 市场

**P - Positioning（定位陈述）**：
> "对注重健康的都市白领而言，我们是新茶饮中坚持'0反式脂肪酸+当日鲜果现切'的品牌，因为所有产品公开热量表且支持自定义糖度，让每一杯都喝得安心。"

### 案例2：企业级AI工具定位

**背景**：某初创公司开发了一款AI辅助代码生成工具，面向开发者群体。

**S - Segmentation（市场细分）**：
- **企业规模**：初创公司（<50人）、中型企业（50-500人）、大型企业（500人以上）
- **技术栈**：前端开发、后端开发、全栈开发、DevOps工程师
- **使用场景**：快速原型开发、遗留代码重构、单元测试生成、代码审查辅助

**T - Targeting（目标选择）**：
选择**中型企业的后端开发团队用于遗留代码重构**场景，理由：
1. 中型企业有足够预算但无法承担大型咨询公司费用
2. 后端遗留系统维护成本高，痛点明确
3. 相比前端，后端代码逻辑复杂，AI辅助价值更大
4. 决策链条短（CTO可直接拍板），销售周期可控

**P - Positioning（定位陈述）**：
> "对中型科技企业的后端技术负责人而言，我们是AI编程工具中专注'Java/C++遗留系统现代化改造'的解决方案，因为能自动识别代码坏味道并生成符合企业规范的重构建议，将重构效率提升3倍。"

## 二、定位图案例

### 案例1：咖啡市场价格×品质定位图

**背景**：某精品咖啡品牌准备进入一线城市，需分析竞争格局。

**定位图**（横轴=价格从低到高，纵轴=品质从低到高）：

```
        品质高
           │         • %Arabica    • Blue Bottle
           │              (40元)       (45元)
           │
           │   • Manner          • 我们的机会点?
           │   (25元)              (35元,更高性价比)
           │
───────────┼────────────────────────────→ 价格高
           │
           │ • 瑞幸            • 库迪
           │ (15元)            (12元)
           │
        品质低
```

**竞品打点分析**：
- **左下角（低价低质）**：瑞幸、库迪，靠补贴和规模化获客
- **左上角（低价高质）**：Manner，主打"小店模式+高品质"，但门店覆盖有限
- **右上角（高价高质）**：%Arabica、Blue Bottle，强调品牌调性和第三空间
- **右下角（高价低质）**：空白，无玩家占据

**空白地分析**：
我们选择**中高端价位（35元）+ 高品质**象限，填补"Manner与%Arabica之间的空白"，主张"用低于国际大牌的价格，获得同等品质的精品咖啡"，通过直营店+外卖结合，避免重资产投入。

### 案例2：项目管理工具功能×易用性定位图

**背景**：某SaaS团队计划推出新一代项目管理工具。

**定位图**（横轴=易用性从差到好，纵轴=功能从少到多）：

```
        功能多
           │   • Jira              • Monday.com
           │   (功能强大但复杂)     (功能全+中等易用)
           │
           │                       • 我们的机会点?
           │                       (功能适中+极简上手)
           │
───────────┼────────────────────────────→ 易用性好
           │
           │ • Trello            • Notion(项目模块)
           │   (简单但功能有限)    (灵活但需自建模板)
           │
        功能少
```

**竞品打点分析**：
- **Jira**：功能最全，但学习曲线陡峭，小团队难以驾驭
- **Monday.com**：功能丰富且可视化好，但价格偏高
- **Trello**：极简看板，适合轻量级任务，缺乏甘特图/资源管理
- **Notion**：灵活性强，但项目管理需自行搭建，非开箱即用

**空白地分析**：
我们切入**"功能适中+极简上手"**象限，提供预设的敏捷开发模板（Scrum/Kanban）、一键生成的甘特图和资源负载视图，目标用户是50-200人的互联网团队，他们觉得Jira太重、Trello太轻。核心价值："3分钟上手，30天落地敏捷流程"。

## 三、交叉SWOT（TOWS）完整案例

### 案例：某在线教育平台的TOWS四组合策略推导

**背景**：某K12在线教育机构完成B轮融资，需制定下一年战略。先梳理SWOT：

**优势（S）**：
- S1：拥有行业顶尖的教研团队，课程完课率高达85%（行业平均60%）
- S2：自研AI学习引擎，能精准诊断学生薄弱点
- S3：品牌在家长群体中口碑良好，NPS达45

**劣势（W）**：
- W1：获客成本高，CAC达800元/人
- W2：产品线单一，仅覆盖初中数学
- W3：技术团队规模小，迭代速度慢于竞品

**机会（O）**：
- O1：政策鼓励素质教育，编程/思维训练赛道增长迅速
- O2：下沉市场在线教育渗透率不足30%，潜力大
- O3：AI大模型技术成熟，可降低内容生产成本

**威胁（T）**：
- T1：头部平台（猿辅导/作业帮）价格战激烈
- T2：监管趋严，广告合规风险增加
- T3：经济下行，家长教育支出缩减

**TOWS四组合策略推导**：

| 组合 | 策略类型 | 具体策略 |
|------|---------|---------|
| **SO**（优势×机会） | 进攻策略 | **策略1**：利用S1教研能力+S2 AI引擎，快速拓展O1编程/思维课程，6个月内上线3门新课，抢占素质教育红利<br>**策略2**：借助S3品牌口碑+O3 AI技术，开发"AI助教"功能降低师资依赖，以低成本进入O2下沉市场 |
| **WO**（劣势×机会） | 改善策略 | **策略3**：针对W2产品线单一，借O1政策东风拓展学科，同时用O3 AI生成内容降低研发成本，解决W3技术团队小的问题<br>**策略4**：针对W1获客成本高，利用O2下沉市场的低竞争环境，通过地推+社群运营降低CAC至500元 |
| **ST**（优势×威胁） | 防御策略 | **策略5**：面对T1价格战，强调S1高完课率和S3高NPS，打出"效果付费"承诺（未提分退款），避开纯价格竞争<br>**策略6**：针对T2监管风险，利用S2 AI引擎实现个性化推荐而非广告投放，降低合规风险 |
| **WT**（劣势×威胁） | 规避策略 | **策略7**：面对W1高CAC+T3消费缩减，暂停大规模品牌投放，转向老带新裂变（奖励机制），控制营销预算<br>**策略8**：针对W3技术弱+T1竞品迭代快，暂缓自研底层技术，采用开源方案+外包非核心模块，集中资源打磨核心AI引擎 |

**战略优先级**：优先执行SO策略（进攻）和ST策略（防御），同步启动WO策略中的产品拓展，WT策略作为风险控制底线。

## 四、定位陈述写作公式+范例

### 定位陈述公式

> **对【目标客户】而言，我们是【品类/框架】中【独特价值主张】的选择，因为【支撑证据/差异化理由】。**

**四要素拆解**：
1. **目标客户**：越具体越好，避免"所有人"
2. **品类/框架**：明确所属赛道，便于对标
3. **独特价值主张**：一句话说明你解决什么痛点/带来什么收益
4. **支撑证据**：数据、技术、模式等可验证的理由

### 3个范例

**范例1（ToC消费品）**：
> "对**每天通勤超过1小时的都市上班族**而言，我们是**即饮咖啡品牌**中**唯一采用冷萃锁鲜技术、开瓶即享现磨口感**的选择，因为**专利保鲜包装能让咖啡风味保持72小时不衰减**。"

**范例2（ToB SaaS）**：
> "对**员工规模100-500人的跨境电商企业**而言，我们是**ERP系统**中**专注多平台订单自动化处理、减少80%人工对账工作**的选择，因为**已对接Amazon/Shopee等12个主流平台API，且支持自定义财务规则引擎**。"

**范例3（平台型产品）**：
> "对**二三线城市的独立设计师品牌**而言，我们是**时尚电商平台**中**提供从生产供应链到内容营销一站式赋能**的选择，因为**自有柔性工厂支持50件小单快反，且签约了200+小红书穿搭博主资源**。"

**常见错误**：
- ❌ "我们是最好的XX平台"（空洞，无差异化）
- ❌ "服务于所有需要XX的用户"（目标客户过于宽泛）
- ❌ "因为我们技术领先"（未说明技术如何转化为用户价值）

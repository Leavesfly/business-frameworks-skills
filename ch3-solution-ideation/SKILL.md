# 第 3 章 · 思索课题的解决方法（Solution Ideation）

**定位**：小循环② 的发散端。在问题被明确定义之后，用发散工具先扩张创意空间，再用收敛工具筛出候选方案。

**与其他章的关系**：
- ← `ch1-problem-discovery`（问题已清晰）
- ⇄ `ch4-strategy-formulation`（小循环②：创意 ↔ 策略）

---

## 框架清单（22–31）

### ㉒ 头脑风暴（Brainstorming）
- **用途**：团队快速发散产生大量想法。
- **四条铁律**：
  1. 不批评
  2. 欢迎异想天开
  3. 越多越好
  4. 组合与改进他人想法
- **步骤**：
  1. 明确课题（一句话写在白板顶部）
  2. 限时 15–25 分钟，人人必发言
  3. 所有想法上墙，不讨论
  4. 分组归类（亲和图）
  5. 投票选出 Top N
- **模板**：
  ```
  课题：___
  ┌─ 想法池 ────────────────┐
  │ #1 ...  #2 ...  #3 ...   │
  │ #4 ...  #5 ...  #6 ...   │
  └──────────────────────────┘
  归类：
  - 主题 A：#1, #3, ...
  - 主题 B：#2, #5, ...
  Top 3：___
  ```
- **英文关键词**：brainstorming, divergent thinking, affinity diagram

### ㉓ 曼陀罗九宫格（Mandala / Lotus Blossom）
- **用途**：围绕中心主题做 8 方向强制联想，再对每个子主题继续展开。
- **步骤**：
  1. 3×3 中心格写核心课题
  2. 围绕它写 8 个相关关键词
  3. 每个关键词作为新 3×3 的中心，再展开 8 个
  4. 最终得到 1 + 8×8 = 65 个点子
- **模板**：
  ```
  ┌─────┬─────┬─────┐
  │  A  │  B  │  C  │
  ├─────┼─────┼─────┤
  │  H  │核心 │  D  │
  ├─────┼─────┼─────┤
  │  G  │  F  │  E  │
  └─────┴─────┴─────┘
  （每一格再展开一个 3×3）
  ```
- **英文关键词**：mandala chart, lotus blossom, 9 grid ideation

### ㉔ 形态分析法（Morphological Analysis）
- **用途**：把方案拆成若干维度，每个维度列出所有选项，组合出新方案。
- **步骤**：
  1. 列出方案的关键维度（如：形态 / 材料 / 动力 / 使用场景）
  2. 每个维度列出 3–6 种可能取值
  3. 跨维度交叉组合 → 产生新方案
- **模板**：
  ```
  | 维度      | 选项 1 | 选项 2 | 选项 3 | 选项 4 |
  |-----------|--------|--------|--------|--------|
  | 形态      |        |        |        |        |
  | 材料      |        |        |        |        |
  | 动力      |        |        |        |        |
  | 场景      |        |        |        |        |
  ```
- **英文关键词**：morphological analysis, parameter matrix, Zwicky box

### ㉕ 脚本法（Scenario Planning）
- **用途**：针对未来不确定性，写出 2–4 个可能发生的"未来剧本"并预演应对。
- **步骤**：
  1. 找出 2 个最关键且最不确定的外部变量
  2. 2×2 交叉产生 4 个情景
  3. 为每个情景写一段叙事
  4. 针对每个情景列对策
- **模板**：
  ```
         变量X ↑
            │
    情景 2  │  情景 1
  ──────────┼──────────→ 变量Y
    情景 3  │  情景 4
            │
  ```
- **英文关键词**：scenario planning, future scenarios, 2x2 scenario matrix

### ㉖ 奥斯本核验表（Osborn Checklist / SCAMPER）
- **用途**：用 9 个提问强制改造既有事物产生新创意。
- **九问**（括号内为 SCAMPER 对应字母）：
  1. 能否**挪作他用**？（Put to other use）
  2. 能否**借用**其他领域？（Adapt）
  3. 能否**修改**颜色/声音/含义？（Modify）
  4. 能否**放大**更大/更多/更强？（Magnify）
  5. 能否**缩小**更小/更轻/更省？（Minify）
  6. 能否**替代**人/材料/过程？（Substitute）
  7. 能否**重组**顺序/模式？（Rearrange）
  8. 能否**颠倒**正反/上下？（Reverse）
  9. 能否**组合**两个合一？（Combine）
- **SCAMPER 七字诀**：S-Substitute / C-Combine / A-Adapt / M-Modify(Magnify/Minify) / P-Put to other use / E-Eliminate / R-Reverse(Rearrange)
- **模板**：
  ```
  | 提问        | 当前事物 | 改造创意 |
  |------------|---------|---------|
  | 挪作他用    |         |         |
  | 借用        |         |         |
  | 修改        |         |         |
  | 放大/缩小   |         |         |
  | 替代        |         |         |
  | 重组        |         |         |
  | 颠倒        |         |         |
  | 组合        |         |         |
  ```
- **英文关键词**：Osborn checklist, SCAMPER, idea triggers, lateral thinking

### ㉗ 创意单（Idea Sheet）
- **用途**：把一个创意写成标准化一页纸，方便比较。
- **模板**：
  ```
  创意名称：
  一句话描述：
  解决谁的什么问题：
  核心机制（怎么做）：
  独特价值：
  所需资源：
  风险：
  下一步验证动作：
  ```
- **英文关键词**：idea sheet, idea canvas, one-pager

### ㉘ 分镜图（Storyboard）
- **用途**：把方案以用户视角的分镜画面讲清楚。
- **步骤**：4–8 格，每格一张图 + 一句旁白，呈现用户从"遇到问题 → 使用方案 → 得到结果"的完整故事。
- **模板**：
  ```
  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
  │ 画面 1  │ │ 画面 2  │ │ 画面 3  │ │ 画面 4  │
  │ 旁白：  │ │ 旁白：  │ │ 旁白：  │ │ 旁白：  │
  └─────────┘ └─────────┘ └─────────┘ └─────────┘
  ```
- **英文关键词**：storyboard, user scenario, narrative prototype

### ㉙ 优缺点表（Pros / Cons List）
- **用途**：对单个候选方案做利弊清单。
- **模板**：
  ```
  方案：___
  ┌───────────────┬───────────────┐
  │ 优点 Pros      │ 缺点 Cons      │
  ├───────────────┼───────────────┤
  │ +             │ -             │
  │ +             │ -             │
  └───────────────┴───────────────┘
  缓解缺点的方法：
  ```
- **英文关键词**：pros and cons, T-chart

### ㉚ SUCCESs（让创意具有黏性的 6 原则）
- **用途**：检验一个创意/信息是否"令人难忘"。
- **六原则**（源自 _Made to Stick_）：
  - **S**imple 简单
  - **U**nexpected 意外
  - **C**oncrete 具体
  - **C**redible 可信
  - **E**motional 情感
  - **S**tories 故事
- **模板**：
  ```
  | 原则       | 当前创意表现 | 改进方向 |
  |------------|--------------|---------|
  | Simple     |              |         |
  | Unexpected |              |         |
  | Concrete   |              |         |
  | Credible   |              |         |
  | Emotional  |              |         |
  | Stories    |              |         |
  ```
- **英文关键词**：SUCCESs model, made to stick, sticky idea

### ㉛ 报酬矩阵（Payoff Matrix / Impact-Effort）
- **用途**：把候选创意按"收益 × 成本"定位，挑"高收益低成本"先做。
- **模板**：
  ```
           高收益 ↑
              │
    快赢区   │  重点投入区
    （先做） │  （规划做）
  ───────────┼──────────→ 高成本
    放弃区   │  审慎评估区
              │
  ```
- 四象限命名可按团队语境调整（如 Quick Wins / Big Bets / Fill-ins / Time Wasters）。
- **英文关键词**：payoff matrix, impact effort matrix, ICE scoring

## 常用组合

- **零想法** → `头脑风暴`㉒ + `奥斯本核验表`㉖
- **想法太散** → `曼陀罗九宫格`㉓ → `形态分析法`㉔
- **方案需要讲故事** → `分镜图`㉘ + `SUCCESs`㉚
- **多个方案要筛选** → `创意单`㉗（标准化）→ `优缺点表`㉙ → `报酬矩阵`㉛
- **面对不确定未来** → `脚本法`㉕

## 输出契约

完成本章后必须产出：
1. **≥ 10 个粗放创意**
2. **3–5 个筛选后的候选方案**（每个一张创意单 ㉗）
3. **报酬矩阵定位图**
4. **建议下一步**：进入 `ch4-strategy-formulation` 把候选方案落地成策略
